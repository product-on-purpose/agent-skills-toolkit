# Backlog

Local-first, version-controlled backlogs (Standard sec 7.1). Two distinct files,
added when the first item is recorded:

- `new-components.md` - proposals to add a skill, command, subagent, hook, etc.
  Each item records: proposed `name`, component type, target `tier`,
  `agent-targets`, rationale, and status. New proposals pass a "why gate".
- `enhancements.md` - features, fixes, and refinements to existing components.
  Each item references the target component and describes the change.

Two backlogs are RECOMMENDED at Bronze and REQUIRED at Silver+. Empty in Phase 0.
