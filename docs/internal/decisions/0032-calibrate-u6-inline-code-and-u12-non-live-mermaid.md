# 0032 - Calibrate U6 (reference-links) for inline code and U12 (mermaid-valid) for non-live diagrams

## TL;DR
- **Decision:** The batch-2 C1 triage's Finding 5. Two precision calibrations, both under one principle - **a check validates only LIVE content, not authored illustrations of syntax**. (1) **U6** (reference-links) now strips single-backtick `` `inline code` `` spans in addition to the fenced ``` / `~~~` blocks ADR 0030 already stripped, so a markdown link or capture regex written as inline code (`` `[text](path)` ``, `` `[^'"]+` ``) is no longer parsed as a dangling reference. The strip is restricted to a single line so an unbalanced stray backtick cannot swallow a following line's real link, and it can only ever REMOVE link matches, never add them. (2) **U12** (mermaid-valid) now skips two non-live block shapes: a block whose body is purely `{{TEMPLATE_PLACEHOLDER}}` token lines (a generation-time slot, never a rendered diagram), and a block commented out inside `<!-- ... -->` (which does not render). Both keep `objective` provenance and still catch real defects; spine stays 29, Standard stays 0.11.
- **Why:** The C1 triage (`levnikolaevich/claude-code-skills`, four MCP-heavy plugins) showed that, after ADR 0030, the residual errors clustered in two evaluator-precision gaps: U6 was blind to inline code (skill docs routinely show `[text](path)` link examples and regex patterns as inline code), and U12 validated `{{...}}` template slots and HTML-commented example diagrams as if they were live. Same legitimacy test as ADR 0028/0029/0030/0031: a check graded against a plugin you do not own must proxy a real, portable defect, not flag an authored illustration.
- **Scope and honesty:** On the four C1 plugins these two fixes take the residual error count from **43 to 12** (documentation-pipeline 20 -> 2, agile-workflow 12 -> 9, codebase-audit-suite 7 -> 1, community-engagement 4 -> 0). They clear the unambiguous "inside markdown code / a template slot / a comment" class. The remaining 12 are a separate, murkier class deliberately NOT auto-suppressed (see Consequences): bare placeholder links (`[Epic Title](link-or-path)`), double-quoted example links (`"See [Manual: library-vX](docs/manuals/...)"`), regexes inside shell single-quotes (`grep '\[.*\](.*\.md)'`), and generator-output-path references - plus exactly one GENUINE link-rot defect (agile `ac_validation_rules.md` via a broken `../../` prefix) that U6 correctly keeps catching. A prior triage note estimated "43 -> 1"; that over-generalized from a few inline-code samples and is corrected here to the measured "43 -> 12".
- **Status:** Accepted.

- **Date:** 2026-06-10

## Context and problem statement
After ADR 0030 cleared U6's fenced-example and non-http-scheme false positives and U11's managed-connector pattern, the four C1 plugins that still errored were re-graded under `--profile plain-plugin` against the current gate. The residuals (identical counts to the original run, so untouched by ADR 0030) decomposed into two precise mechanisms:

- **U6 inline-code blindness.** `scripts/checks/reference-links.mjs` stripped ``` and `~~~` fences but not single-backtick inline code. The `LINK` regex then matched links and link-shaped regexes written as inline code: `` `[file.md](link)` `` (a formatting how-to), `` `[text](path)` `` (a "how to extract links" example), and capture regexes such as `` `import\s+...(['"])([^'"]+)['"]` `` (which the LINK regex reads as `](­[^'"]+)`). These are authored examples, not live references.
- **U12 non-live diagrams.** `scripts/checks/mermaid-valid.mjs` validated every fenced ```mermaid block, including (a) `{{BUSINESS_CONTEXT_DIAGRAM}}` template slots that the documentation generator substitutes at run time (7 of the 8 U12 errors), and (b) an erDiagram EXAMPLE kept inside an `<!-- ... -->` comment (the 8th, failing on the crow's-foot `||--o{` cardinality glyphs). Neither is a live, rendered diagram.

## Decision drivers
- **Legitimacy:** a link inside inline code, a `{{...}}` template slot, and a commented-out diagram are not portable defects; erroring on them fails well-built docs on authoring convention.
- **Completion over divergence:** U6 already strips fences for exactly this reason; inline code is the same idea one step further.
- **One-directional safety:** stripping code/comments and skipping placeholder/commented blocks can only REMOVE findings, never add them, so the calibration cannot newly break any passing target; the only risk is a false negative, and a link inside code or a diagram in a comment is non-live by definition.
- **Do not over-fit:** the murkier residual class (bare placeholder links, quoted examples, shell regexes, output-path references) is left alone because detecting it needs content-meaning heuristics (is `link` a placeholder or a file named "link"? is `.*\.md` a regex or a filename?) that would risk suppressing the one genuine link-rot defect in the same corpus.
- **No conformance churn:** the toolkit's own Gold self-grade stays Advanced 0/0; spine and Standard are untouched.

## Considered options
1. **Strip inline code in U6; skip template-placeholder and HTML-commented blocks in U12.** (chosen) Surgical, principled, one-directional-safe; clears the unambiguous class. TDD: failing tests first (a link/regex in inline code ignored, a real link OUTSIDE code still caught, a stray backtick not swallowing the next line; a pure `{{...}}` block skipped, a mixed placeholder+non-keyword block still validated, a commented block skipped, a live block after a commented bad one still validated), then minimal code.
2. **Also auto-suppress the murkier residuals** (placeholder-word targets, quoted-string examples, regex-metacharacter targets, output paths). Rejected: each needs a content heuristic with false-negative risk; the corpus contains a genuine `../../` link-rot defect that such heuristics could mask. A few false positives on heavily-templated docs is acceptable signal; hiding a real broken link is not.
3. **Reclassify U6/U12 to house and drop them under plain-plugin.** Rejected (the ADR 0029 move): both proxy real, portable defects (a genuinely dangling reference; a malformed live diagram). The problem is precision, not provenance.
4. **Leave as-is.** Rejected: the inline-code and template/comment classes were the dominant residual false positives after ADR 0030, and they are exactly the friction the corpus exists to find.

## Decision outcome
Option 1.
- `scripts/checks/reference-links.mjs`: `stripFences` becomes `stripCode`, which strips fenced blocks then inline-code spans before the LINK scan. The inline match is `` /(`+)(.+?)\1/g `` - a run of N backticks closed by the next same-length run (CommonMark), so a link in a double- or triple-backtick example is stripped too; `.` excludes line terminators, keeping the single-line restriction so a stray backtick cannot cross a newline. (The multi-backtick handling was a pre-merge adversarial-review refinement; the first cut matched only single backticks.)
- `scripts/checks/mermaid-valid.mjs`: a **fence-aware**, line-preserving `stripHtmlComments` runs before block extraction, and an `isTemplatePlaceholder` guard skips a block whose non-blank body lines are all lone `{{...}}` tokens. The comment handling neutralizes only a comment that BEGINS OUTSIDE a code fence (a commented-out fence): a `<!-- ... -->`-looking span INSIDE a live ```mermaid block is diagram source the renderer receives verbatim and is left intact, so U12 always validates exactly what renders. Fence tracking follows CommonMark closer rules (the same marker, length at least the opener, and only whitespace after), so a body line that merely starts with ``` does not falsely close a live block. (Both the fence-awareness and the strict closer were pre-merge adversarial-review findings - the first cut stripped comments globally, and a second cut used a loose fence tracker; either would have let malformed content hidden in a comment-span inside a LIVE diagram escape validation.)
- Tests: `tests/unit/reference-links.test.mjs` (+6: inline-code link ignored, inline-code regex ignored, real link outside code still flagged, stray-backtick safety, double-backtick span ignored, triple-backtick span ignored) and `tests/unit/mermaid-valid.test.mjs` (+7: pure placeholder skipped, indented placeholder skipped, mixed placeholder still validated, commented block skipped, live-after-commented still validated, a comment-span inside a LIVE block still validated, and a non-closing fence-like line does not desync the comment skip). Suite 375 -> 388; gate Advanced 0/0; spine 29; Standard 0.11.
- Verified against the C1 corpus: 43 residual errors -> 12, as broken down in the TL;DR; the genuine agile link-rot and the murkier example/output-path class remain (the latter by design).

## Consequences
- **Positive:** the gate stops flagging link and regex examples written as inline code, `{{...}}` template slots, and commented-out diagrams; U6's code handling now matches its own fence rationale; outward grading under plain-plugin is materially quieter on heavily-documented plugins without losing a real defect.
- **Negative / accepted:** U6 no longer flags a dangling link that exists only inside inline code (acceptable: it is an illustration), and U12 no longer validates a diagram that is a template slot or commented out (acceptable: neither renders). A genuinely malformed LIVE erDiagram that uses crow's-foot `||--o{` cardinality is a SEPARATE latent question (U12's bracket counter does not model erDiagram cardinality) - not triggered by any live diagram in this corpus, recorded as a possible future finding, and out of scope here.
- **Residual, deliberately deferred:** the murkier example-link class (bare placeholder targets like `link` / `link-or-path`, double-quoted example links with literal `vX` / `05` placeholders, regexes inside shell single-quotes, and generator-output-path references such as `docs/manuals/library-vX.md`) is left flagged. Auto-suppressing it needs content heuristics with false-negative risk; the maintainer can revisit it as its own decision if the corpus signal warrants, weighing it against the genuine `ac_validation_rules.md` link-rot the same mechanism correctly catches.
