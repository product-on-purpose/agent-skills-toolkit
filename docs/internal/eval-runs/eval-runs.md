### Toolkit findings from this batch (sensor readings, with proposed disposition)

1. **`evaluate.mjs` component scope silently ignores `--profile`.** Found pre-run: `evaluate.mjs <skill-dir> --profile plain-plugin --json` returns `profile: undefined` and house checks (U5) still fire on a third-party single skill. Plugin scope honors the flag; component scope drops it without a warning. Either apply profiles in component scope or reject the flag there; silent acceptance is the worst option. **Disposition: calibration candidate (ADR + TDD).**
2. **The behavioral mode's `evals/` convention has zero instances anywhere.** Neither the toolkit's 23 skills nor any third-party target ships an `evals/` eval-set; both behavioral runs had to derive ad-hoc case sets. The mode is currently aspirational, and the derive-from-description fallback both graders used is undocumented. Either document the fallback in `askit-evaluate` behavioral mode and `agents/askit-quality-grader.md`, or ship eval-set fixtures (askit-build-samples exists to author them). **Disposition: doc/spec fix candidate.** (Also surfaced independently by R1 as an info finding.)
3. **`agents/askit-reviewer.md` step 2 names `check.mjs` where the targeted `evaluate.mjs` is the right tool** (R1's minor finding, independently confirmed: `check.mjs` has no component scope and grades a bare skill directory as an empty plugin root, U8 "No skills found under skills/", vacuous pass). The reviewer delegate should run `node scripts/evaluate.mjs <target> --json`. **Disposition: doc fix candidate.**
4. **U6 residual false-positive class confirmed on a second corpus plugin.** R2 confirmed both `documentation-pipeline` U6 errors are the known Finding-5 residual class (bare example links in NO_CODE_EXAMPLES prose: `[Component Library](link)`, `[docker-compose.yml](../docker-compose.yml)`). Strengthens the ADR 0032 residual dataset; still deliberately not auto-suppressed (the same class hides genuine link-rot). **Disposition: data point for the existing Finding-5 residual decision; no action now.**
5. **U5 scorer corpus data for the deferred recalibration.** Both Anthropic skills (mcp-builder, pdf) cluster at exactly 0.65; R3 traced why: the scorer's action-verb regex is a strict 12-verb list, so 'Guide for creating...' and gerunds ('creating', 'building') miss it. Direct input for the deferred ADR 0029 follow-up. **Disposition: feeds the U5 recalibration TDD set.**
6. **`askit-build-skill` chain-vs-procedure gap (R4 case B4, a real fail).** The frontmatter `chain` declares `askit-skill-author` delegation and the Scope section asserts it, but neither create nor improve mode steps ever instruct the delegation - a literal execution does the work inline. Add an explicit delegate step or state the subagent is an optional implementation detail. **Disposition: skill improvement candidate.**
7. **Behavioral summary semantics are underspecified.** The `summary` block (`fired`/`missed`/`behaviorPass`/`behaviorFail`) has no documented rule for where a failed *no-fire* trigger case counts (R4 filed its T7 no-fire fail under `behaviorFail`); the schema in `docs/reference/evaluation-reports.md` names the fields but not their semantics. Define: `fired` = should-fire cases that fire, `missed` = should-fire cases that do not, and add a false-fire counter or fold no-fire fails explicitly. **Disposition: doc/schema fix candidate.**
