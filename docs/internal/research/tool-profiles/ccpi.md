# ccpi (claude-code-plugins-plus-skills) - profile

ccpi is the CLI package manager (`@intentsolutionsio/ccpi`, installable via `pnpm add -g @intentsolutionsio/ccpi`) for a community-maintained, open-source marketplace of Claude Code plugins and Agent Skills hosted at `tonsofskills.com` and published at `github.com/jeremylongshore/claude-code-plugins-plus-skills`. Authored by Jeremy Longshore at Intent Solutions, the repository catalogs 425-447 plugins and roughly 2,810-3,023 skills (the two primary sources give slightly different snapshots; see Notes), validates them against the AgentSkills.io specification plus Anthropic's Claude Code references using a deterministic 100-point Python rubric, and assigns badge tiers (Gold/Silver/Bronze). The grading system applies to every plugin in the catalog, not only to inbound PRs.

## Dimensions

1. **Unit of evaluation:** per-plugin (the verification pipeline grades every individual plugin in the catalog; each plugin's skills are scored and the plugin receives a badge tier). [source](https://github.com/jeremylongshore/claude-code-plugins-plus-skills/blob/main/scripts/run-verification-pipeline.mjs) (Confidence: High)
2. **Tiered and climbable:** score with badge tiers - Gold (90-100), Silver (75-89), Bronze (60-74), None (<60); a letter-grade scale (A/B/C/D/F) also exists for PR pre-screen commentary, but the durable artifact written to `marketplace.extended.json` is the Gold/Silver/Bronze/None badge assigned by score threshold. This is a score with named tiers derived from it, not a burndown checklist. [source](https://github.com/jeremylongshore/claude-code-plugins-plus-skills/blob/main/scripts/run-verification-pipeline.mjs) (Confidence: High)
3. **Verdict basis:** hybrid - the primary scoring in `validate-skills-schema.py` is fully deterministic (rule-based Python: field presence, line counts, structural checks, keyword patterns, no LLM call); the PR pre-screen workflow additionally calls `summarize.py` which invokes a DeepSeek LLM for advisory summaries, but that LLM path is explicitly marked advisory-only and never drives the merge verdict. Deterministic rules are the authoritative gate. [source](https://github.com/jeremylongshore/claude-code-plugins-plus-skills/blob/main/.github/workflows/pr-prescreen.yml) and [source](https://github.com/jeremylongshore/claude-code-plugins-plus-skills/blob/main/scripts/validate-skills-schema.py) (Confidence: High)

Deprecation is not documented as a formal lifecycle stage; governance beyond maintainer review is not documented. ## Notes

- **Score vs. blocking gate discrepancy.** The 100-point rubric is widely marketed as the merge gate ("C-grade rejects merge"), but `pr-prescreen.yml` explicitly declares itself "Advisory only. Never a required check. Failures here NEVER block merges." The actual blocking CI jobs in `validate-plugins.yml` enforce catalog integrity, schema shape, lint, and security patterns, not the full 100-point score. The score is informative and drives badge assignment, but does not deterministically prevent a merge. This is a material difference from how the tool is described on the site, and a sharp contrast with a gate whose exit code blocks CI.
- **Catalog count volatility.** The GitHub README (2026-06-10) reports 425 plugins / 2,810 skills; tonsofskills.com (2026-06-10) reports 447 plugins / 3,023 skills. Both are live counters; both recorded as dated snapshots.
- **LLM in the pipeline.** `summarize.py` calls the DeepSeek API for advisory PR summaries (hybrid in the advisory flow only); the deterministic `validate-skills-schema.py` is the sole scorer for badge assignment.
- **npm page inaccessible.** `https://www.npmjs.com/package/@intentsolutionsio/ccpi` returned HTTP 403; download statistics (519 cited in README) could not be independently verified from the npm primary source.
- **No should-trigger or behavioral evals.** Despite being a skill ecosystem, there is no published framework for verifying a skill activates under correct context triggers; testing is limited to schema validation and marketplace UI.
- **daily-skill-generator.yml.** A workflow generating skills automatically exists, raising a provenance/quality-floor question the rubric does not cleanly separate.
