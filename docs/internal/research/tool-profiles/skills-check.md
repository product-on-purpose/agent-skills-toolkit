# skills-check - profile

skills-check is a 14-command CLI and GitHub Action quality toolkit for Agent Skills SKILL.md files, authored by Chris Williams (voodootikigod) of Vercel and published at `github.com/voodootikigod/skills-check` (MIT license, 2026 copyright). It covers version-drift detection, security auditing (hallucinated packages, prompt injection, dangerous commands, URL liveness), metadata linting, token budget analysis, semver-bump verification, organizational policy enforcement, and behavioral eval execution. The GitHub Action (`voodootikigod/skills-check@v1`, marketplace page: `github.com/marketplace/actions/skills-check-ci`) is the primary CI delivery surface; the `skills-check` npm package is also installable as a global CLI. The tool describes itself as "the missing quality toolkit for Agent Skills" and targets the Agent Skills open specification at agentskills.io. [source](https://github.com/voodootikigod/skills-check) [source](https://skillscheck.ai) [source](https://github.com/marketplace/actions/skills-check-ci)

## Dimensions

1. **Unit of evaluation:** per-skill. Each SKILL.md file is individually evaluated by commands such as `audit`, `lint`, `budget`, `verify`, and `test`. The `health` command aggregates across a directory, but individual skill files are the unit of analysis. [source](https://skillscheck.ai/docs) [source](https://github.com/voodootikigod/skills-check/blob/main/README.md) (Confidence: High)

2. **Tiered and climbable:** pass-fail with configurable severity thresholds. There is no tiered badge or burndown framework. Each command has a `--fail-on` threshold (e.g., `audit-fail-on: high`; `lint-fail-on: error`; `policy-fail-on: blocked`) and exit codes 0 (pass) / 1 (findings at or above threshold) / 2 (config error). The `health` command combines audit + lint + budget + policy into a single pass-fail CI gate. No multi-tier ladder with labelled advancement levels is documented. [source](https://github.com/marketplace/actions/skills-check-ci) [source](https://skillscheck.ai/docs) (Confidence: High)

3. **Verdict basis:** hybrid - deterministic core with optional LLM enhancement. `audit`, `lint`, `budget`, `policy`, `fingerprint`, `fix`, `doctor`, and `check` are fully deterministic (rule-based, no LLM). `verify` runs a three-path heuristic classifier (structural diff, package diff, content similarity) independently and optionally merges in an LLM classification via `combineClassification()`; the `--skip-llm` flag disables LLM entirely and the heuristic path still produces a verdict. `test` uses LLM for rubric grading of eval suites; `refresh` uses LLM to propose skill content updates. The authoritative security and linting gates are deterministic. [source](https://github.com/voodootikigod/skills-check/blob/main/packages/cli/src/verify/index.ts) [source](https://skillscheck.ai/docs) (Confidence: High)

4. **Lifecycle coverage:** author (init, lint, fix) + validate (audit, lint, budget, health, doctor) + version (verify, fingerprint) + release (doctor, CI integration). The `init` command scaffolds a `skills-check.json` registry from a skills directory. `lint --fix` fills missing frontmatter from git context. `verify` validates that declared semver bumps match content diffs. `doctor` checks release-readiness prerequisites. `refresh` proposes LLM-assisted content updates when product versions drift. Deprecation and formal governance lifecycle are not documented in any primary source reviewed. [source](https://skillscheck.ai/docs) [source](https://github.com/voodootikigod/skills-check) (Confidence: Medium - lifecycle coverage inferred from the command inventory; deprecation/governance absence is absence of evidence, not confirmed absence)

5. **Cross-agent emission:** none. skills-check is an auditing and validation tool for existing SKILL.md files; it does not generate or emit skills in any format. No multi-format transpilation, no Codex `.codex-plugin/` emission, no Gemini CLI format output. The README and docs make no mention of cross-agent format generation. [source](https://github.com/voodootikigod/skills-check/blob/main/README.md) [source](https://skillscheck.ai/docs) (Confidence: High)

6. **Self-proving / dogfooded:** not confirmed. The CI workflow (`ci.yml`) runs `pnpm install`, `pnpm audit --prod`, `pnpm run build`, `pnpm run lint`, `pnpm run typecheck`, and `pnpm run test` on the monorepo. These are standard development checks, not invocations of the skills-check CLI against the project's own skills. No `.github/workflows/` job running `skills-check health` or `skills-check audit` against the repo's own content was found in the primary source inspection. [source](https://github.com/voodootikigod/skills-check/blob/main/.github/workflows/ci.yml) (Confidence: High - CI workflow read directly; self-check invocation is absent, not merely not found by search)

7. **Target spec(s):** agentskills.io base spec. The tool's README and official site both describe it as "a quality and integrity layer for Agent Skills" and link to `agentskills.io`. The frontmatter schema validated (`name`, `description`, `compatibility`) is the Agent Skills baseline format. The spec itself was originally developed by Anthropic and released as an open standard; governance is community/Anthropic per the agentskills.io site and the agentskills/agentskills repository (not AAIF, consistent with the adjudication in `METHODOLOGY.md`). No Claude Code-extended, Codex, or Gemini-specific formats are targeted. [source](https://github.com/voodootikigod/skills-check/blob/main/README.md) [source](https://agentskills.io) (Confidence: High)

8. **Validation depth:** strict-CI + frontmatter-quality + refs + budget. `audit` performs six-tier content checking: hallucinated package detection against npm/PyPI/crates.io plus advisory database integration (Aikido Security, Socket.dev); prompt injection pattern detection; dangerous command scanning (`rm -rf`, `curl | bash`); URL liveness via HEAD requests with 10s timeout; and metadata completeness validation. `lint` checks frontmatter field presence and structural quality. `budget` measures per-section token counts and detects redundancy, with snapshot comparison for tracking changes. `fingerprint` generates SHA-256 content hashes and watermarks for tamper detection. `health` combines audit + lint + budget + policy in a single blocking gate. [source](https://github.com/voodootikigod/skills-check/blob/main/README.md) [source](https://skillscheck.ai/docs) (Confidence: High)

9. **Versioning support:** semver-enforced with LLM-or-heuristic diff verification. The `verify` command is the key mechanism: it reads the declared version from skill frontmatter (`compatibility` or legacy `product-version` fields), retrieves the prior version from git history via `getPreviousVersion()` (or explicit `--before`/`--after` paths), computes the declared bump level (major/minor/patch) via semver comparison, then runs three parallel heuristic analyses (structural diff, package diff, content similarity) combined optionally with LLM classification to assess whether the actual content change warrants that bump level. A mismatch between declared and assessed bump level is a verification failure. The `--suggest` flag proposes the appropriate bump level. `--skip-llm` gives a fully deterministic heuristic-only path. `fingerprint` adds content hashes and watermarks for runtime provenance. [source](https://github.com/voodootikigod/skills-check/blob/main/packages/cli/src/verify/index.ts) [source](https://skillscheck.ai/docs) (Confidence: High)

10. **Eval / test scaffolding:** yes - eval suites with rubric grading, regression detection, and budget caps. The `test` command executes skill eval test suites from `tests/` directories. Supported test types: trigger, outcome, style, regression. Harnesses: `claude-code` and `generic` (custom shell template). Rubric-based grading is performed by an LLM (Anthropic, OpenAI, or Google); provider and model are configurable. Budget caps (`--max-cost` in dollars) prevent runaway costs. Regression detection compares current results against stored baselines (`--update-baseline` to accept new results; `--ci` exits 1 on regressions). Multi-trial execution (`--trials`) with pass thresholds (`--pass-threshold`). The `test` command runs external shell commands via agent harnesses; the docs flag this as requiring container isolation for untrusted skills (`--isolation`; v1.2.0 added Isolation Zones for Docker/Podman sandboxing). [source](https://github.com/voodootikigod/skills-check/blob/main/packages/cli/src/commands/test.ts) [source](https://github.com/marketplace/actions/skills-check-ci) [source](https://skillscheck.ai/docs) (Confidence: High)

11. **Security posture:** multi-layer. The `audit` command scans for prompt injection patterns, dangerous shell commands, hallucinated package references, and broken URLs. SECURITY.md declares responsible disclosure (reports to voodootikigod@gmail.com, 72-hour response), CodeQL static analysis on PRs, SHA-pinned CI actions kept current by Dependabot, and Dependabot automated dependency monitoring. The `test` command's security boundary is explicitly documented: external shell execution via agent harnesses is flagged as requiring container isolation. In-scope vulnerability classes include command injection, supply chain risks in the CLI/Action, prompt injection in LLM-assisted commands, and auth bypasses. The `--allow-unsafe-local` flag is required in CI to override isolation, providing an explicit opt-in surface. [source](https://github.com/voodootikigod/skills-check/blob/main/SECURITY.md) [source](https://github.com/voodootikigod/skills-check/blob/main/README.md) (Confidence: High)

12. **Output formats:** human (Chalk-colored terminal with severity icons) + JSON (machine-readable full report structure) + markdown (summary table + severity sections, PR-ready) + SARIF (SARIF 2.1.0 for GitHub Security tab). All four formats apply to `audit`; `terminal` and `json` apply to `verify`, `lint`, `fix`, `doctor`; `terminal`, `json`, and `markdown` apply to `test`, `budget`, `usage`. The GitHub Action exposes a `report` output (markdown) and a `results` output (JSON with per-command exit codes). [source](https://github.com/voodootikigod/skills-check/blob/main/README.md) [source](https://skillscheck.ai/docs) (Confidence: High)

## Notes

- **verify is the standout dimension.** The three-path heuristic analysis (structural diff + package diff + content similarity) combined with optional multi-provider LLM classification for semver bump validation is architecturally more sophisticated than metadata-only approaches. The `--skip-llm` fallback ensures it remains usable without API keys, making it both powerful and CI-safe. This is likely the strongest differentiating feature relative to other tools in this comparison.

- **test command is broad but carries execution risk.** The `test` command's ability to run arbitrary shell commands via agent harnesses, with rubric grading by LLM, baseline regression detection, multi-trial execution, and cost caps, is a comprehensive behavioral eval scaffold. The explicit security boundary documentation (isolation required, `--allow-unsafe-local` for CI override) is honest about the risk surface. The v1.2.0 addition of Isolation Zones (Docker/Podman) shows active mitigation. This is not a lightweight "smoke test" - it is a full eval harness.

- **Rapid release cadence in March 2026.** All 9 formal releases landed between March 2 and March 7, 2026 - a 5-day sprint from v0.1.0 to v1.2.1. Subsequent main-branch work (94+ commits, June 2026) has not produced a new formal release tag as of the verification snapshot. This means the marketplace action's v1.2.1 may be significantly behind the current main branch.

- **Not dogfooded.** The CI workflow does not run skills-check against the project's own skills or content. For a tool whose primary value claim is quality assurance for Agent Skills, the absence of self-application is notable. Compare: the agent-skills-toolkit project runs its own gate against itself in CI as a documented spine check.

- **Star count (9) vs. scope mismatch.** The tool covers 14 commands spanning version drift, multi-source security auditing, behavioral evals, token budgeting, policy enforcement, and SARIF output - a scope comparable to or exceeding well-starred tools in adjacent spaces. A star count of 9 as of the verification snapshot indicates the tool is either very new, not yet publicly promoted, or both. This is consistent with the March 2026 initial release date. No independent verification of adoption metrics (downloads, active users) was possible from primary sources.

- **LLM providers supported.** `verify`, `test`, and `refresh` all support Anthropic, OpenAI, and Google as providers (configurable via `--provider` and `--model`). The `@ai-sdk/anthropic`, `@ai-sdk/openai`, and `@ai-sdk/google` packages are optional peer dependencies, installed separately by the user. This is a multi-provider design, not locked to one LLM vendor.

- **Audit caching is a CI-performance differentiator.** Package lookups cache for 1 hour (SHA-256 filenames in `~/.cache/skills-check/audit/`); URL checks cache for 12 hours with retry-next-run on failure; in-memory cache caps at 1000 items with concurrent request merging. This level of caching design is more sophisticated than typical linting tools and suggests the tool was built with large skill libraries in mind.

- **`skillscheck.ai/docs/policy` returned 404.** The policy format schema was assembled from the policy source code (`packages/cli/src/policy/index.ts`) and the skillscheck.ai/docs index summary, rather than a dedicated docs page. The structural fields (sources, banned, metadata, content, freshness; severity levels blocked/violation/warning) are confirmed from primary source code, but a complete `.skill-policy.yml` example was not retrievable from a docs page.

- **Vercel association.** The author's GitHub profile lists Vercel as employer. The tool validates against the Agent Skills spec, not a Vercel-specific format. However, the `vercel-labs/skills` repository (separate from this tool) had an open PR (#509) from voodootikigod adding a `skills validate` command, suggesting the author is working across the Vercel skills ecosystem and this tool may have a closer Vercel upstream relationship than the personal-account hosting implies.
