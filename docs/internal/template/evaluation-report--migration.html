01Executive summary

The skills work; they just are not packaged as a library. Migration is mostly mechanical, and the first stage gets you a graded Bronze in about an afternoon.

acme-content-kit is four content skills and one command sitting in a hand-organized folder for a single agent (Claude Code). They run, but nothing declares them a library: there is no library.json at the root, the components are not normalized to agentskills.io anatomy, names and prefixes are inconsistent, and there is no manifest or cross-agent emission. Because the deterministic gate keys off library.json and the generated manifests, it has nothing to grade - hence "loose components," not a tier.

The good news is that the gap is packaging, not quality. The skill bodies are within budget and their descriptions are usable. Recommended path: run askit-migrate to adopt a library.json and normalize anatomy (Stage 1 -> Bronze), then add multi-agent emission and generated manifests (Stage 2 -> Silver), then wire the advanced spine - a documented hook, self-hosting, release automation, and docs frontmatter (Stage 3 -> Gold). Each stage ends at a real, gradeable tier, so you can stop at any landing.

02What was assessed

A candidate library: a directory of components that has not yet been declared or structured as a plugin. The assessment inventories what is present and maps it against the tier requirements.

  • Skills (4): blog-post, writeNewsletter, seo_audit, content-calendar - naming is mixed (kebab, camel, snake) and unprefixed.
  • Command (1): a /draft command with no documented contract.
  • Root: no library.json, no INDEX.md, no scripts/, no generated manifests/.
  • Targets: a raw .claude/ layout only - nothing emits for Codex or other agentskills.io agents.
  • Hygiene: bodies are within the instruction budget; two descriptions are thin; no em or en dashes found.

03Methodology & scope

A migration assessment is a gap analysis, not a grade. It runs the same requirement catalog the evaluator uses, but reads each requirement as "present / absent / to add" against the three tiers, then sequences the absent ones into the cheapest order that reaches each landing.

  • Inventory. Enumerate the components and the root layout, infer intended names and roles.
  • Gap by tier. For Bronze (Universal), Silver (Convergent), and Gold (Advanced), mark each requirement present or to-add, with the evidence found.
  • Sequence. Order the to-add work so every stage ends at a gradeable tier and earlier work is reused, not redone.

06Insights

◆
The distance is packaging, not rewriting. The skills already pass the content checks (budget, house style); what is missing is the library scaffolding around them. That makes Bronze cheap and the whole climb low-risk - you are not reworking behavior.
◆
library.json is the keystone. Until it exists the gate cannot grade anything, so every other gap is downstream of Stage 1. Do that first and the report flips from "loose components" to a real tier in one move.
◆
Every stage is a landing. Bronze, Silver, and Gold are each shippable. If you only need cross-agent emission, stop at Silver; the advanced spine is optional until you want the Gold guarantees.
