# Changelog

All notable changes to this project are documented here. The format is based on
[Keep a Changelog](https://keepachangelog.com/en/1.1.0/), and this project adheres
to [Semantic Versioning](https://semver.org/spec/v2.0.0.html).

`CHANGELOG.md` is the full technical history. A curated, user-facing
`RELEASE-NOTES.md` is added at Gold (Standard sec 10.6); the two are distinct.

## [Unreleased]

### Added
- **Demonstrative hook (`hooks/`): the toolkit now ships one real hook, so Gold check `G1` (hook-documentation) grades a real artifact instead of passing vacuously.** A `PreToolUse` guard on `Write|Edit|NotebookEdit` (`hooks/no-dashes.mjs`) denies any payload whose `new_string` / `content` / `new_source` contains an em-dash (U+2014) or en-dash (U+2013), emitting an actionable substitution message (replace with ' - ' or restructure); clean or unreadable payloads are allowed (a hook bug never wedges the session). It enforces the Standard's `U10` no-dash rule at author-time. Documented per sec 3.5 in `hooks/README.md` (event/trigger/matcher/scope/failure) and covered by a `G3` regression eval (`evals/no-dashes-hook.eval.json`, `covers: { "hook": "PreToolUse" }`). The script references the banned characters by character code so it stays `U10`-clean itself. This is **P0** of the ADR 0024 / `plan_v1.1.0` build-out (it sits on `main` unreleased; it reaches installers only at the next release tag). P0 adds no new check, so the spine stays at 25; it makes the existing `G1` non-vacuous. Gate green at Advanced, 211 tests.

## [1.0.0] - 2026-06-02

The first tagged Gold release: the toolkit becomes installable through the `product-on-purpose` marketplace. It folds in everything since `v0.2.0` - the full v1 builder catalog, the Gold self-conformance gate, and full Astro site conformance (clauses 14.1-14.11) - plus the launch changes below.

### Changed
- **Repositioned the README around the plugin lifecycle - start, grow, govern, and level up an advanced cross-agent plugin - instead of reading as a skill builder, and made the tier model scannable** (a six-column "at a glance" table plus tight per-tier cards in place of the dense inline-ID prose). The repositioning keeps an explicit beginner on-ramp alongside the advanced maintainer track, and softens the "zero-dependency" overclaim to the accurate "one runtime dependency, a YAML parser" (matching `package.json` and the docs site).
- **`library.json` now declares `standard: "0.9"`** (was `0.8`), adopting the v0.9 Standard the repo amended in `STANDARD.md` (runner Node baseline `>=22.12.0`; ADR 0025). Version bumped `0.3.0` -> `1.0.0` for the marketplace launch; the native manifests, `manifest.generated.json`, and `INDEX.md` were regenerated from `library.json` (the version source of truth).
- **Astro site toolchain and branding converged to clauses 14.6/14.8 and 14.2 (full 14.1-14.11 conformance).** CI (`ci.yml`, `deploy-pages.yml`) now resolves Node via `node-version-file: .nvmrc` so the committed pin is the single CI source of truth, instead of a hardcoded `node-version: "24"` in all three `setup-node` steps; `actions/checkout` and `actions/setup-node` move to `@v5` (the Pages actions were already `@v5`). The mermaid theme is branded with `mermaidConfig.themeVariables` (`lineColor: '#5C7CFA'`, system-ui, 14px), mirroring the pm-skills reference. With the new 14.11 guards below, the site satisfies every Astro site clause. See ADR 0026 (full Astro site conformance: local 14.11 guards, node-version-file, branded mermaid).
- **The Advanced Skill Library Standard is amended to v0.9: the runner Node baseline rises from EOL Node 20 to `>=22.12.0` (recommended pin Node 24).** Node 20 reached end-of-life (2026-04-30) and is below Astro 6's floor (the basis of every family documentation site), so the published `>=20` baseline pointed downstream plugins at an unsupported runtime. STANDARD.md sec 4.1, the engines example, and the resolution log are updated; see ADR 0025. The toolkit's own `library.json` adopts `standard: "0.9"` in this 1.0.0 release (see the standard-bump entry above); the Standard sets a floor, so a plugin requiring stricter remains conformant.
- **Astro docs site converged to the family site standard (site clauses 14.1-14.10).** Dropped end-of-life Node 20 family-wide: `engines.node >=22.12.0` in `library.json`, root `package.json`, and `site/package.json`, with committed `.nvmrc` and `.node-version` pinning Node 24 (Active LTS). CI and the GitHub Pages deploy now build on Node 24, and the Pages actions are pinned to `actions/upload-pages-artifact@v5` + `actions/deploy-pages@v5`. Added a non-deploying `build-site` job to `ci.yml` so a PR proves the Astro build compiles; a `catalog-coverage` drift test (`tests/unit/catalog-coverage.test.mjs`) that keeps the hand-authored catalog page (`site/src/content/docs/catalog.md`) honest with `library.json` (every component named, no orphan `askit-` name) without flattening its editorial prose; and `site/public/robots.txt` pointing at the auto-emitted sitemap. Made the splash hero action and the `overview.md` intra-site links base-relative so the GitHub Pages base lives only in `astro.config.mjs` (clause 14.7). The site already satisfied Pattern S and the stock `docsLoader()`. Deferred to the planned shared `@product-on-purpose/astro-docs-preset`: a default `og:image`.
- **The toolkit now declares `tier: advanced` (Gold) and self-validates at Gold.** The full gate (Bronze U1-U11, Silver S1-S8, Gold G1-G7) is green and `tier-report` reports `advanced` with an empty burndown - a non-vacuous claim backed by real checks (the same discipline used at Silver). Version bumped `0.2.0` -> `0.3.0` (the Phase 4 full catalog + the Phase 5 Gold gate). `INDEX.md` regenerated for the new tier/version; README, AGENTS.md, and the conformance docs reframed Silver -> Gold.

## [0.2.0] - 2026-05-28

### Changed
- The toolkit now declares and satisfies Silver: `library.json` `tier: convergent` with a `components` index (closes S3). All Convergent checks (S1-S6) gate green; `tier-report` reports `convergent` with an empty burndown. Version bumped to 0.2.0.

### Added
- Validation spine (Node, zero test-framework deps): `loadPlugin` context, `Finding` model, and a `yaml`-backed frontmatter parser.
- Eight Universal (Bronze) conformance checks (U1-U8): library.json schema, AGENTS.md presence, skill frontmatter validity, name-equals-directory, description scoring (0.7 warn), reference-link resolution, instruction-budget warning, and native-manifest drift.
